API DESIGN을 들여다보다

2017년 5월 18일

API DESIGN

생각해보니 내가 API를 제대로 만들어 본 적이 없었더랬다.

그럼 REST는 설명할 수 있던가? HTTP는?

GraphQL이 재미있어 보이니 써보고는 싶은데, 인증을 붙인 프로덕션 레벨의 코드를 만들어 본 적이 없다.

그럼 해야지.

배움에 있어서, 지금까지는 발표를 통해 드라이브를 걸었다면, 이제 블로그에 정리하면서 익혀보려고 한다

‘정확히 알기’ 그 첫번째 과정으로 API 디자인에 도전해본다.

이 시리즈가 끝날 즈음이면 나는

  • API 서버를 구축해서 인증을 붙여본다
  • 전통적인 RESTful API와 GraphQL 서버 모두 구현
  • REST에 대해 제대로 설명하게 된다
  • 여러 인증 과정을 경험해본다

동시에

  • HTTP 완벽가이드 릴레이 세미나
    • HTTP 완벽가이드로 적당히 완벽히 알기(아하…금새 또 적당히 적당히)
    • 그림으로 배우는 시리즈로 네트워크와 HTTP에 대한 쉬운 설명을 익히기

어떻게?

  • 검색과 블로그와 책으로부터 얻은 지식을 에버노트에 거칠게 정리한 다음
  • 블로그에 옮겨 담는다
